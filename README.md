# ShareWatcher — монитор сетевой папки

Следит за общей сетевой папкой (шарой) и решает три задачи:

1. **Нарушения прав доступа** — сравнивает реальные права на папки с текущим
   членством пользователей в группах Active Directory и сигнализирует о любых
   несоответствиях.

2. **Слишком длинные пути** — обнаруживает файлы и папки, путь к которым
   приближается к лимиту Windows в 256 символов или превысил его, и
   предупреждает об этом (создаёт файл-предупреждение в проблемной папке).

3. **Устаревшие файлы** — находит файлы, которые давно не открывались,
   и при необходимости переносит их в архив.

---

## Требования

| Что нужно | Примечание |
|---|---|
| Windows Server 2016 / 2019 / 2022 | Или Windows 10/11 в домене |
| PowerShell 5.1 | Уже установлен на всех современных версиях Windows |
| Запуск от имени администратора | Нужен для чтения ACL |
| RSAT — средства Active Directory | Только для анализа прав (`-Scan`). Установить: Параметры > Приложения > Дополнительные компоненты > «Средства RSAT: доменные службы Active Directory» |

---

## Файлы

```
ShareWatcher.ps1   Основной скрипт
config.json        Конфигурация (отредактировать перед первым запуском)
```

Оба файла должны лежать в одной папке на сервере.

---

## Быстрый старт

```powershell
# 1. Разрешить запуск скриптов (один раз, от администратора)
Set-ExecutionPolicy RemoteSigned

# 2. Указать путь к шаре в config.json, затем запустить полную проверку
.\ShareWatcher.ps1 -Scan

# 3. Поставить на автозапуск каждую ночь в 02:00
.\ShareWatcher.ps1 -Setup

# 4. Запустить мониторинг длины путей в реальном времени
.\ShareWatcher.ps1 -Watch
```

Перед запуском скрипт проверяет, что шара доступна, и пишет в лог
контрольную сумму (SHA256) используемого конфига — по логу всегда видно,
с какими настройками выполнялся каждый прогон.

## Команды

### `-Scan`

Обходит всю шару и формирует отчёт по трём направлениям.

**Нарушения прав доступа** (требует RSAT):

| Тип | Что означает |
|---|---|
| `UnauthorizedAccess` | Пользователь имеет прямую запись в ACL папки, но не состоит ни в одной группе с доступом к ней. Классика: пользователя перевели в другую группу, а старые прямые права не убрали. Если запись даёт права на изменение — уровень Critical. |
| `OrphanedSID` | Учётная запись удалена из AD, но её SID остался в правах папки. Учётные записи компьютеров (кластеры, агенты резервного копирования) распознаются и ложно не помечаются. |
| `DisabledUserHasAccess` | Отключённая учётная запись имеет явную запись в ACL. |
| `DisabledUserInActiveGroup` | Отключённый пользователь всё ещё числится в группе с доступом к шаре. |
| `MemberOfMultipleTopGroups` | Пользователь одновременно в нескольких основных группах шары (скорее всего, не был исключён из старой). Учитываются группы с правами как на папках первого уровня, так и на самом корне шары. |
| `RedundantDirectAccess` | Пользователь уже имеет доступ через группу, но у него ещё и прямая запись в ACL — при переводе в другую группу она останется. |
| `EmptyGroupHasAccess` | Группа указана в ACL папки, но в ней нет ни одного пользователя. |
| `InheritanceDisabled` | Папка вручную изолирована от родительских прав. |

Права на запись определяются по битовым флагам записи в ACL (включая
generic-права), несколько записей для одного пользователя на одной папке
объединяются. Вложенное членство в группах раскрывается на стороне
контроллера домена одним LDAP-запросом (правило
LDAP_MATCHING_RULE_IN_CHAIN) — без ограничения на размер группы и без
риска зацикливания на вложенных группах.

**Длина путей** — перечисляет все пути, близкие к лимиту или превысившие его.
В каждой проблемной папке создаётся файл-предупреждение (имя настраивается,
по умолчанию `!PATH_LENGTH_WARNING.txt`). За один прогон создаётся не более
`MaxWarningFiles` таких файлов.

**Устаревшие файлы** — список файлов, не открывавшихся дольше `StaleFileDays`
дней.

Отчёт выводится в консоль и сохраняется в папку `LogDirectory`.

```powershell
.\ShareWatcher.ps1 -Scan
.\ShareWatcher.ps1 -Scan -Fix    # то же + перемещение устаревших файлов в архив
```

При архивации файлы, открытые пользователями в момент переноса,
пропускаются с пометкой в логе и отчёте и переносятся при следующем запуске.

### `-Watch`

Работает непрерывно и отслеживает создание новых файлов и папок на шаре.

Если у созданного файла или папки путь приближается к лимиту (`MaxPathLength`)
или уже превысил его, в родительской папке создаётся (или обновляется)
файл-предупреждение с пояснением и информацией о том, сколько символов
осталось для новых имён. Файл автоматически удаляется, когда проблема с
длиной пути в этой папке исчезает.

```powershell
.\ShareWatcher.ps1 -Watch
```

### `-Setup`

Регистрирует задачу в Планировщике Windows: каждый день в 02:00 запускается
`-Scan` от имени учётной записи SYSTEM (с флагом `-Fix`, если в конфиге
включено `AutoArchive`). Задача выполняется с политикой RemoteSigned.

```powershell
.\ShareWatcher.ps1 -Setup
```

Расписание можно изменить в Планировщике задач > `ShareWatcher`.

### Дополнительные параметры

```powershell
# Разово проверить другую шару, не трогая config.json (удобно для тестов)
.\ShareWatcher.ps1 -Scan -SharePath \\OTHERSERVER\Share

# Использовать альтернативный файл конфигурации
.\ShareWatcher.ps1 -Scan -Config .\config-test.json
```

## config.json

```json
{
    "SharePath": "\\\\SERVER\\Share",
    "MaxPathLength": 256,
    "WarnAtRemaining": 30,
    "StaleFileDays": 180,
    "ArchiveFolderName": "_Archive",
    "AutoArchive": false,
    "LogDirectory": "C:\\ShareWatcher\\Logs",
    "WarningFileName": "!PATH_LENGTH_WARNING.txt",
    "MaxWarningFiles": 500,
    "ADSearchBase": "",
    "ExcludeFolders": [
        "System Volume Information",
        "$RECYCLE.BIN"
    ],
    "IgnoreGroups": [
        "Domain Admins",
        "Enterprise Admins",
        "Schema Admins",
        "Backup Operators"
    ]
}
```

Обязателен только `SharePath`. Любой другой ключ можно не указывать —
подставится значение по умолчанию. Пустая строка в `ADSearchBase`
означает «весь домен» (эквивалентно отсутствию ключа).

| Параметр | Что делает | По умолчанию |
|---|---|---|
| `SharePath` | UNC-путь к шаре. **Обязательно** (можно переопределить параметром `-SharePath`). | — |
| `MaxPathLength` | Жёсткий лимит длины пути в символах | 256 |
| `WarnAtRemaining` | Предупреждать, когда для имени файла остаётся меньше N символов | 30 |
| `StaleFileDays` | Файл считается устаревшим через столько дней без обращения | 180 |
| `ArchiveFolderName` | Имя папки-архива внутри шары | `_Archive` |
| `AutoArchive` | Перемещать устаревшие файлы при запуске `-Scan -Fix` (и в задаче из `-Setup`). Принимает `true`/`false`, а также строки `"true"`, `"1"`, `"yes"`. | false |
| `LogDirectory` | Куда сохранять логи и отчёты | рядом со скриптом, в подпапке `Logs` |
| `ExcludeFolders` | Имена папок, которые полностью пропускаются при обходе | системные |
| `IgnoreGroups` | Группы AD, которые не включаются в анализ прав | — |
| `WarningFileName` | Имя файла-предупреждения о длине пути | `!PATH_LENGTH_WARNING.txt` |
| `MaxWarningFiles` | Максимум файлов-предупреждений, создаваемых за один прогон | 500 |
| `ADSearchBase` | DN контейнера (OU), которым ограничиваются все запросы к AD. Заметно ускоряет загрузку в больших доменах. Пример: `"OU=Groups,DC=corp,DC=local"` | весь домен |

## Решение проблем

**Скрипт не запускается: «невозможно загрузить файл»**
```powershell
Set-ExecutionPolicy RemoteSigned
```

**Ошибки при запуске (Unexpected token, Missing brace)**
Файл повреждён при передаче (проблема кодировки). Скачайте `ShareWatcher.ps1`
заново и скопируйте на сервер напрямую, не открывая в браузере.

**«Config file not found» или «'SharePath' is required»**
Скрипт запускается из папки, где нет `config.json`, либо в конфиге не задан
путь к шаре. Перейдите в папку скрипта или укажите конфиг явно:
`-Config C:\ShareWatcher\config.json`. Путь к шаре можно передать и без
конфига: `-SharePath \\SERVER\Share`.

**«Share is not available»**
Шара по указанному пути недоступна с этой машины. Проверьте путь в конфиге,
сетевую доступность сервера и права учётной записи, от которой запущен скрипт.

**«Модуль ActiveDirectory не найден»**
Установите RSAT на машине, где запускается скрипт (см. раздел «Требования»).
Без него анализ прав по группам AD не работает — обнаруживаются только
`OrphanedSID`, проверка длины путей и устаревших файлов работает как обычно.

**«Нет доступа» к некоторым папкам**
Запускайте скрипт от учётной записи администратора домена или учётной записи
с правом чтения всех папок шары.

**«Could not expand group ...» в логе**
Не удалось получить членов конкретной группы AD (нет прав или проблема
связи с контроллером домена). Скрипт продолжает работу; членство этой
группы в анализе считается пустым, поэтому возможны ложные срабатывания
`UnauthorizedAccess` по её участникам — проверьте причину в тексте ошибки.

**Загрузка AD идёт слишком долго**
В большом домене выгрузка всех пользователей и групп может занимать
десятки минут. Задайте `ADSearchBase` в конфиге, чтобы ограничить запросы
контейнером, где находятся группы доступа к шаре.

**Сколько длится полное сканирование?**
Загрузка AD: 1–3 минуты (в больших доменах дольше — см. `ADSearchBase`).
Обход папок: примерно 2–5 минут на каждые 10 000 папок.
