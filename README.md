# ShareWatcher — монитор сетевой папки

Следит за общей сетевой папкой (шарой) и решает три задачи:

1. **Нарушения прав доступа** — сравнивает реальные права на папки с текущим
   членством пользователей в группах Active Directory и сигнализирует о любых
   несоответствиях.

2. **Слишком длинные пути** — обнаруживает файлы и папки, путь к которым
   приближается к лимиту Windows в 256 символов, предупреждает пользователей
   в реальном времени и автоматически удаляет папки, превысившие лимит.

3. **Устаревшие файлы** — находит файлы, которые давно не открывались,
   и при необходимости переносит их в архив.

---

## Требования

| Что нужно | Примечание |
|---|---|
| Windows Server 2016 / 2019 / 2022 | Или Windows 10/11 в домене |
| PowerShell 5.1 | Уже установлен на всех современных версиях Windows |
| Запуск от имени администратора | Нужен для чтения ACL и удаления папок |
| RSAT — средства Active Directory | Только для анализа прав (`-Scan`). Установить: Параметры > Приложения > Дополнительные компоненты > «Средства RSAT: доменные службы Active Directory» |

---

## Файлы

```
ShareWatcher.ps1   Основной скрипт
config.json        Конфигурация (отредактировать перед первым запуском)
```

Оба файла должны лежать в одной папке на сервере.

---

## Быстрый старт

```powershell
# 1. Разрешить запуск скриптов (один раз, от администратора)
Set-ExecutionPolicy RemoteSigned

# 2. Указать путь к шаре в config.json, затем запустить полную проверку
.\ShareWatcher.ps1 -Scan

# 3. Поставить на автозапуск каждую ночь в 02:00
.\ShareWatcher.ps1 -Setup

# 4. Запустить мониторинг длины путей в реальном времени
.\ShareWatcher.ps1 -Watch
```

## Команды

### `-Scan`

Обходит всю шару и формирует отчёт по трём направлениям.

**Нарушения прав доступа** (требует RSAT):

| Тип | Что означает |
|---|---|
| `UnauthorizedAccess` | Пользователь имеет прямую запись в ACL папки, но не состоит ни в одной группе с доступом к ней. Классика: пользователя перевели в другую группу, а старые прямые права не убрали. |
| `OrphanedSID` | Учётная запись удалена из AD, но её SID остался в правах папки. |
| `DisabledUserHasAccess` | Отключённая учётная запись имеет явную запись в ACL. |
| `DisabledUserInActiveGroup` | Отключённый пользователь всё ещё числится в группе с доступом к шаре. |
| `MemberOfMultipleTopGroups` | Пользователь одновременно в нескольких основных группах шары (скорее всего, не был исключён из старой). |
| `RedundantDirectAccess` | Пользователь уже имеет доступ через группу, но у него ещё и прямая запись в ACL — при переводе в другую группу она останется. |
| `EmptyGroupHasAccess` | Группа указана в ACL папки, но в ней нет ни одного пользователя. |
| `InheritanceDisabled` | Папка вручную изолирована от родительских прав. |

**Длина путей** — перечисляет все пути, близкие к лимиту или превысившие его.
В каждой проблемной папке создаётся файл `!PATH_LENGTH_WARNING.txt`.

**Устаревшие файлы** — список файлов, не открывавшихся дольше `StaleFileDays` дней.

Отчёт выводится в консоль и сохраняется в папку `LogDirectory`.

```powershell
.\ShareWatcher.ps1 -Scan
.\ShareWatcher.ps1 -Scan -Fix    # то же + перемещение устаревших файлов в архив
```

### `-Watch`

Работает непрерывно и отслеживает создание новых файлов и папок.

**Создана папка с длинным путём:**
- Папка автоматически **удаляется**.
- В родительской папке создаётся `!PATH_LENGTH_WARNING.txt` с объяснением
  и информацией о том, сколько символов осталось для новых имён.

**Создан файл с длинным путём:**
- Файл **не удаляется** (там может быть работа пользователя).
- В родительской папке создаётся `!PATH_LENGTH_WARNING.txt`.

Если в `config.json` включено `UserNotifications.Enabled`, пользователю
дополнительно отправляется всплывающее окно прямо на его экран.

```powershell
.\ShareWatcher.ps1 -Watch
```

### `-Setup`

Регистрирует задачу в Планировщике Windows: каждый день в 02:00 запускается
`-Scan` от имени учётной записи SYSTEM.

```powershell
.\ShareWatcher.ps1 -Setup
```

Расписание можно изменить в Планировщике задач > `ShareWatcher`.

### `-SetupClient`

Выводит инструкцию по установке клиентского наблюдателя на каждый ПК
пользователей (подробнее — в разделе «Уведомления» ниже).

```powershell
.\ShareWatcher.ps1 -SetupClient
```

## config.json

```json
{
    "SharePath": "\\\\SERVER\\Share",
    "MaxPathLength": 256,
    "WarnAtRemaining": 30,
    "StaleFileDays": 180,
    "ArchiveFolderName": "_Archive",
    "AutoArchive": false,
    "LogDirectory": "C:\\ShareWatcher\\Logs",
    "ExcludeFolders": ["System Volume Information", "$RECYCLE.BIN"],
    "IgnoreGroups": ["Domain Admins", "Enterprise Admins"],
    "Email": { "Enabled": false, "..." : "..." },
    "UserNotifications": { "Enabled": false, "..." : "..." }
}
```

| Параметр | Что делает | По умолчанию |
|---|---|---|
| `SharePath` | UNC-путь к шаре. **Обязательно.** | — |
| `MaxPathLength` | Жёсткий лимит длины пути в символах | 256 |
| `WarnAtRemaining` | Предупреждать, когда для имени файла остаётся меньше N символов | 30 |
| `StaleFileDays` | Файл считается устаревшим через столько дней без обращения | 180 |
| `ArchiveFolderName` | Имя папки-архива внутри шары | `_Archive` |
| `AutoArchive` | Перемещать устаревшие файлы при запуске `-Scan -Fix` | false |
| `LogDirectory` | Куда сохранять логи и отчёты | рядом со скриптом |
| `ExcludeFolders` | Имена папок, которые полностью пропускаются при обходе | системные |
| `IgnoreGroups` | Группы AD, которые не включаются в анализ прав | — |

### Настройки уведомлений пользователям

```json
"UserNotifications": {
    "Enabled": true,
    "MessageTitle": "ShareWatcher - предупреждение о длине пути",
    "AutoDiscoverComputer": false,
    "DiscoveryTimeoutSeconds": 3,
    "ComputerMap": {
        "ivanov": "PC-IVANOV01",
        "petrov": "PC-PETROV02"
    }
}
```

| Параметр | Что делает |
|---|---|
| `Enabled` | Включить или выключить всплывающие уведомления |
| `MessageTitle` | Заголовок окна уведомления |
| `ComputerMap` | Таблица `имя_пользователя -> имя_компьютера`. Имя пользователя: `echo %USERNAME%`. Имя компьютера: `hostname`. |
| `AutoDiscoverComputer` | Если true — автоматически ищет компьютер пользователя по всему домену. Медленнее, требует WMI-доступа к клиентам. |

## Уведомления пользователям

Когда в режиме `-Watch` обнаруживается нарушение длины пути, ShareWatcher
пытается показать сообщение на экране пользователя, который создал проблемный
объект.

### Как работает доставка

1. ShareWatcher читает владельца (NTFS Owner) нового файла или папки —
   это и есть пользователь, который его создал.
2. Записывает файл уведомления в `\\шара\_Notifications\<имя_пользователя>.txt`.
3. Пытается доставить сообщение немедленно через WMI (запускает `msg.exe`
   локально на клиенте) или через `msg.exe /SERVER:` как запасной вариант.
4. Клиентский скрипт-наблюдатель проверяет файл уведомления каждые 10 секунд
   и вызывает `msg.exe *` локально — без каких-либо сетевых разрешений.

### Установка на клиентский ПК (один раз на каждой машине)

Запустите на сервере:

```powershell
.\ShareWatcher.ps1 -SetupClient
```

Скрипт выведет два блока:

**Шаг 1** — текст небольшого скрипта. Сохраните его в файл
`C:\ShareWatcher_Notify.ps1` на клиентском ПК.

**Шаг 2** — команда для создания задачи планировщика на клиенте
(выполнять от администратора):

```cmd
schtasks /create /tn ShareWatcher_Notify /tr "powershell.exe -WindowStyle Hidden -NoProfile -File C:\ShareWatcher_Notify.ps1" /sc onlogon /rl limited /f
```

Задача запускается тихо при каждом входе пользователя и показывает
всплывающее окно в момент, когда сервер записывает уведомление.

## Решение проблем

**Скрипт не запускается: «невозможно загрузить файл»**
```powershell
Set-ExecutionPolicy RemoteSigned
```

**Ошибки при запуске (Unexpected token, Missing brace)**
Файл повреждён при передаче (проблема кодировки). Скачайте `ShareWatcher.ps1`
заново и скопируйте на сервер напрямую, не открывая в браузере.

**«Модуль ActiveDirectory не найден»**
Установите RSAT на машине, где запускается скрипт (см. раздел «Требования»).

**«Нет доступа» к некоторым папкам**
Запускайте скрипт от учётной записи администратора домена или учётной записи
с правом чтения всех папок шары.

**Всплывающие окна не доходят до пользователей**
Смотрите строки `[WARN]` и `[ERROR]` в лог-файле после события обнаружения.
Частые причины:

| Что написано в логе | Что делать |
|---|---|
| `WMI: Access denied` | Скрипт должен запускаться от учётной записи с правами администратора на клиентском ПК |
| `WMI: RPC server unavailable` | Открыть правило брандмауэра на клиенте: `netsh advfirewall firewall set rule group="Windows Management Instrumentation (WMI)" new enable=yes` |
| `msg.exe exit 5` | Включить AllowRemoteRPC на клиенте: `reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v AllowRemoteRPC /t REG_DWORD /d 1 /f` |
| Все методы не работают | Использовать клиентский наблюдатель (`-SetupClient`) — он работает только через SMB (порт 445), который уже открыт, раз шара доступна |

**Сколько длится полное сканирование?**
Загрузка AD: 1–3 минуты. Обход папок: примерно 2–5 минут на каждые 10 000 папок.
